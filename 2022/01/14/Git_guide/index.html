Git 使用指南

Git 分布式版本控制系统

初次运行Git

查看Git配置文件

Git 配置文件存放在三个位置：

  1. /etc/config 用于修改系统上每一个用户和仓库的通用配置。可以使用 git config --system 来读写该文件
  2. ~/.gitconfig 或 ~/.config/git/config 此配置文件只针对于当前用户有效。可以使用 git config --global 来读写该文件
  3. Git仓库中的 .git/config 文件，只针对于仓库有效

配置文件优先级从上到下以此增加。

查看所有配置文件位置及其配置命令: git config --list --show-origin

修改用户信息

在使用 Git 之前，需要设置用户名和密码，在每一次的提交中会记录这两个信息。

获取 Git 仓库

本地创建仓库

在文件目录下运行 git init 便可以在文件夹内生成一共 .git 文件夹，里面包含了 Git 的各种信息

克隆仓库

使用 git clone <url> 来获取一个 Git 仓库的拷贝，这个命令会在当前文件夹下创建一个与仓库名一致的文件夹，文件夹里则是仓库的文件。

自定义克隆文件夹名，可以使用 git clone <url> new_name 命令，这样克隆下来的文件名则为自定义的 new_name

使用 git clone 时可以使用 https 协议和 ssh 协议。

更新仓库文件

在 Git 系统中，一共有四种类型的文件：

  1. Untracked File
  2. Unmodified File
  3. Modified File
  4. Staged File

查看文件状态

可以使用 git status 来查看文件处于何种状态。

可以使用 git status -s 或者 git status --short来查看简洁后的文件状态信息。

当文件夹下新建了一个文件 README.md, git status 会显示出该文件为 Untracked file，意味着 Git 在之前的快照中未出现此文件。

跟踪新文件

使用 git add README 来使 Git 跟踪该文件。使用该命令后文件会变为 Staged file. 如果 git add 的文件为目录，则意味着跟踪该文件夹下的所有文件。

暂存已修改文件

当一个已经被跟踪的文件修改后, git status 会显示该文件 Changes not staged for commit, 表示该文件已经修改但未放入暂存区, 可以使用 git add 将其放入暂存区。若将该文件放入暂存区后又修改了该文件，那么如果想提交最新的更改，则需要再次将其加入暂存区，否则提交的是该文件上一次加入暂存区时的内容。

git add 可以跟踪新文件，也可以将已跟踪的文件加入暂存区。

忽略文件

对于某些不想加入 Git 管理的文件，会总是出现在未跟踪的列表中。此时需要在 Git 项目中合适的文件夹下创建一个名为 .gitignore 的文件。

gitignore 文件规范如下：

  • 所有空行或者以 # 开头的行都会被 Git 忽略。
  • 可以使用标准的 glob 模式匹配，它会递归地应用在整个工作区中。
  • 匹配模式可以以（/）开头防止递归。
  • 匹配模式可以以（/）结尾指定目录。
  • 要忽略指定模式以外的文件或目录，可以在模式前加上叹号（!）取反。

查看修改内容

对于一个已暂存的文件，暂存后又将其修改，可以使用 git diff 命令查看修改的部分。

不加参数命令查看未暂存的文件的修改部分：

git diff

查看已暂存的文件和最后一次提交之间的差异：

git diff --staged

staged 与 staged 是同一个用法：

git diff --cached

提交更新

对于已经暂存的文件，想将其提交，可以使用如下命令：

git commit

将提交信息和命令放在同一行，可以使用如下命令：

git commit -m "message"

若提交时不想使用繁琐的 git add 将文件加入暂存区可以使用如下命令跳过 git add 步骤，将所有已跟踪的文件进行提交：

git commit -a -m "message"

移除文件

删除文件

手动删除文件后，运行 git status 时会将其记录在 Changes not staged for commit 中增加删除该文件的记录。

不再跟踪文件

移除 Git 中的某个文件使其不再被跟踪，可以使用如下命令：

git rm

此后 Git 将不再将该文件纳入版本管理。

删除已修改或已暂存文件

如果文件删除前已修改或已放到暂存区，必须使用如下命令：

git rm -f

保留文件删除

如果想从 Git 中删除，但在目录中保留该文件，即想让 Git 不再跟踪该文件，可以使用如下命令：

git rm --cached xxx

参数中的 xxx 也可以是 glob 模式

移动文件

当在 Git 中重命名了一个文件, Git 也能判断出这是一次改名操作。

在 Git 中对文件进行改名，可以使用如下命令：

git mv old_name new_name

查看提交历史

查看提交历史，可以使用如下命令：

git log

以上命令会将提交按时间从最近提交的时间开始，依次向下展示提交记录。每条 log 会展示出提交的校验和、作者名、邮箱地址、提交时间以及提交说明。

可以使用 -p -n 或者 --patch -n 来查看最近的 n 次提交。

如果想查看提交的简略统计信息，可以使用 --stat 选项，其可以展示每次提交中修改的文件、修改文件数目、修改的方式（增加或删除）的信息。

可以使用 --pretty 开改变提交历史的展示格式, --pretty中有多个选项，包括 oneline short full fuller format, 例如：

git log --pretty=oneline

git log -pretty=format:"%h - %an, %ar : %s"

git log --pretty=format常用选项:

选项 说明
%H 提交的完整哈希值
%h 提交的简写哈希值
%T 树的完整哈希值
%t 树的简写哈希值
%P 父提交的完整哈希值
%p 父提交的简写哈希值
%an 作者名字
%ae 作者电子邮件地址
%ad 作者修订日期
%ar 作者修订日期，按多久以前的方式显示
%cn 提交者名字
%ce 提交者电子邮件地址
%cd 提交日期
%cr 提交日期，距离今天多长时间方式显示
%s 提交说明

git log 常用选项：

选项 说明
-p 按补丁格式显示每个提交引入的差异
–stat 显示每次提交的文件修改统计信息
–shortstat 只显示 –stat 中最后的行数修改添加移除统计
–name-only 仅在提交信息后显示已修改的文件清单
–name-status 显示新增、修改、删除的文件清单
-abbrev-commit 仅显示 SHA-1 校验和所有 40 个字符中的前几个字符
–relative-date 使用较短的相对时间而不是完整格式显示日期
–graph 在日志旁以 ASCII 图形显示分支与合并历史
-pretty 使用其他格式显示历史提交信息
–oneline –pretty=oneline –abbrev-commit 合用的简写

如果想看到的 log 根据时间来筛选，可以使用如下命令：

查看某个含有字符串的添加或删除提交：

git log -S string

比如查看提交文件中含有 sort 关键字的提交记录：

git log -S sort

限制 git log 输出的选项：

选项 说明
- 仅显示最近的 n 条提交
–since,–after 仅显示指定时间之后的提交
–until,–before 仅显示指定时间之前的提交
–author 仅显示作者匹配指定字符串的提交
–committer 仅显示提交者匹配指定字符串的提交
–grep 仅显示提交说明中包含指定字符串的提交
-S 仅显示添加或删除内容匹配指定字符串的提交
